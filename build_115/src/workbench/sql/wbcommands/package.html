Implementation of Workbench specific SQL commands.
A new internal command needs to be registered in two places:
  • {@link workbench.sql.CommandMapper}
  • {@link workbench.sql.wbcommands.CommandTester}

The CommandMapper class is used to find the implementation of a SQL command during execution of SQL script.

The CommandTester is used to identifiy Workbench internal commands e.g. during formatting or for syntax highlighting in the editor. It also drives the list of commands that are listed when running WbHelp in the console.
